# Git Convention

Mandatory standard for all git operations in this repository.
All rules are authoritative.

---

## 1. Branch protection (MANDATORY)

`main` is a protected branch. Direct commits to `main` are forbidden.

Every change — no matter how small — MUST go through a branch and a pull
request. There are no exceptions.

Branch protection is enforced on GitHub:
- Direct push to `main` is blocked.
- Squash merge is the only permitted merge strategy.

---

## 2. Branch naming

All branches MUST follow this pattern:

```
type/<short-name>
```

| Prefix | Use for |
|---|---|
| `feat/` | New capability |
| `fix/` | Bug fix |
| `test/` | Tests only, no production code change |
| `docs/` | Documentation only |
| `refactor/` | Refactor with no behaviour change |
| `chore/` | Build tooling, dependency updates |

`<short-name>` MUST be lowercase, hyphen-separated, and describe the change:

```
feat/script-parser
feat/audio-capture
fix/ring-buffer-boundary
test/whisper-provider
docs/go-logging-convention
chore/add-gitignore
```

Forbidden branch names:
- `main`, `master`, `dev`, `develop`, `staging`
- Generic names: `fix`, `update`, `changes`, `wip`
- Names with uppercase letters or underscores

---

## 3. Commit format

All commits MUST follow Conventional Commits:

```
type(scope): short description

optional body
```

### Subject line rules

- All lowercase
- Maximum 72 characters
- Imperative mood: `add parser` not `added parser`
- No full stop at the end

### Body rules

- Blank line between subject and body
- Explain WHY, not what the diff shows
- Wrap at 72 characters
- Omit when the WHY is obvious from the subject

---

## 4. Types

| Type | Use for |
|---|---|
| `feat` | New feature or capability |
| `fix` | Bug fix |
| `test` | Adding or correcting tests |
| `docs` | Documentation only |
| `refactor` | Code change that is neither a fix nor a feature |
| `chore` | Build process, dependency updates, tooling |

---

## 5. Scopes

Match the package or layer being changed:

| Scope | Package |
|---|---|
| `script` | `internal/script` |
| `audio` | `internal/audio` |
| `stt` | `internal/stt` |
| `llm` | `internal/llm` |
| `tts` | `internal/tts` |
| `loop` | `internal/loop` |
| `tui` | `internal/tui` |
| `session` | `internal/session` |
| `config` | `internal/config` |
| `deps` | `go.mod` / `go.sum` |

---

## 6. Prohibited content

MUST NOT appear in any commit message or PR body:

- The word `Claude`
- The word `AI` or `artificial intelligence`
- Any assistant or tool name
- `Generated by`, `Co-authored by` (except standard git trailers)
- References to conversations, sessions, or prompts

Commits describe the code change. They do not describe how it was produced.

---

## 7. Commit examples

**Correct:**

```
feat(script): add markdown parser for phase and beat metadata
```

```
fix(loop): prevent whisper refire after beat is covered

The refire timer was checked before the state update was applied,
causing a second whisper even when the beat was covered in the same cycle.
```

```
test(audio): add ring buffer boundary tests
```

**Forbidden:**

```
Add script parser as suggested
Fixed the bug
feat: various improvements
update main.go
```
